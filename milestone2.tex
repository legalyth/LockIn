\section{Introduction}

\textbf{Lock-In} is a secure, full-stack Kanban-style task management application. It allows users to register, log in, and manage their tasks across three columns --- \textbf{To Do}, \textbf{Doing}, and \textbf{Done} --- with support for priority levels, due dates, tags, and drag-and-drop reordering.

This document covers the Milestone~2 deliverable: the front-end development progress, updated UI mockups reflecting the actual implementation, and any changes to the project plan.

\subsection{State Management (Pinia)}

Four Pinia stores manage the application state:

\subsubsection{authStore --- Authentication}
\begin{itemize}[nosep]
    \item \textbf{State:} \texttt{token}, \texttt{userId}, \texttt{username}
    \item \textbf{Actions:} \texttt{login()}, \texttt{register()}, \texttt{updateProfile()}, \texttt{logout()}
    \item \textbf{Features:} Token persistence in localStorage, session expiry warning (5~min before), auto-logout on token expiration
\end{itemize}

\subsubsection{taskStore --- Tasks}
\begin{itemize}[nosep]
    \item \textbf{State:} \texttt{tasks[]}, \texttt{loading}, \texttt{error}, \texttt{searchQuery}, \texttt{filterPriority}, \texttt{sortBy}
    \item \textbf{Actions:} \texttt{fetchTasks()}, \texttt{createTask()}, \texttt{updateTask()}, \texttt{deleteTask()}
    \item \textbf{Computed:} \texttt{todoTasks}, \texttt{doingTasks}, \texttt{doneTasks}, \texttt{overdueCount}, \texttt{progressPercent}
    \item \textbf{Features:} Real-time search filtering, priority filtering, sorting (Newest / Due Date / Priority), overdue detection
\end{itemize}

\subsubsection{themeStore --- Dark Mode}
\begin{itemize}[nosep]
    \item \textbf{State:} \texttt{isDark} (persisted in localStorage)
    \item \textbf{Actions:} \texttt{toggle()}, \texttt{apply()}
    \item \textbf{Integration:} Adds/removes \texttt{dark} class on \texttt{<html>} for Tailwind's class-based dark mode
\end{itemize}

\subsubsection{toastStore --- Notifications}
\begin{itemize}[nosep]
    \item \textbf{State:} \texttt{toasts[]}
    \item \textbf{Actions:} \texttt{success()}, \texttt{error()}, \texttt{info()}, \texttt{remove()}
    \item \textbf{Features:} Auto-dismiss after 3 seconds, three visual types (green / red / blue)
\end{itemize}

% ──────────────────────────────────────────────
\subsection{Routing and Navigation Guards}

Vue Router 4 is configured with a \textbf{global navigation guard} that protects authenticated routes:

\begin{lstlisting}
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token');
  if (to.meta.requiresAuth && !token) {
    next('/login');
  } else {
    next();
  }
});
\end{lstlisting}

\begin{itemize}[nosep]
    \item Routes \texttt{/dashboard} and \texttt{/profile} are protected (\texttt{meta: \{ requiresAuth: true \}})
    \item Unauthenticated users are redirected to \texttt{/login}
    \item A catch-all route handles 404 errors
\end{itemize}

% ──────────────────────────────────────────────
\subsection{API Integration Layer}

An Axios instance is configured with:

\begin{itemize}[nosep]
    \item \textbf{Base URL:} \texttt{http://localhost:3000/api}
    \item \textbf{Request interceptor:} Automatically attaches \texttt{Authorization: Bearer <token>} header
    \item \textbf{Response interceptor:} On 401 responses, clears localStorage and redirects to \texttt{/login}
\end{itemize}

\section{Key Features Implemented}

\subsection{Drag-and-Drop Task Management}

Tasks can be dragged between the three columns (To Do, Doing, Done) using the \textbf{vuedraggable} library. When a task is dropped in a new column, its status is automatically updated via the API. A ghost effect (30\% opacity) provides visual feedback during dragging.

\subsection{Real-Time Search and Filtering}

\begin{itemize}[nosep]
    \item \textbf{Search bar:} Filters tasks in real-time by title and description (case-insensitive)
    \item \textbf{Priority filter:} Dropdown to show All / High / Medium / Low priority tasks
    \item \textbf{Sorting:} By Newest, Due Date (earliest first), or Priority (high to low)
\end{itemize}

\subsection{Overdue Detection}

Tasks with a past due date that are not marked as ``Done'' are flagged with:
\begin{itemize}[nosep]
    \item A red border on the task card
    \item An ``Overdue'' badge
    \item Red text on the due date
    \item An overdue counter in the summary banner
\end{itemize}

\subsection{Dark Mode}

Full dark mode support using Tailwind CSS \texttt{class} strategy. The toggle button in the navbar persists the user's preference in localStorage. All components have dark mode variants.

\subsection{Toast Notification System}

A global notification system displays contextual messages:
\begin{itemize}[nosep]
    \item \textbf{Success (green):} Task created, updated, deleted, profile updated
    \item \textbf{Error (red):} API failures, validation errors
    \item \textbf{Info (blue):} Session expiry warnings
\end{itemize}
Messages auto-dismiss after 3 seconds with smooth animations.

\subsection{Session Expiry Management}

\begin{itemize}[nosep]
    \item JWT tokens expire after 24 hours
    \item A warning toast appears 5 minutes before expiry
    \item Auto-logout and redirect to login when token expires
    \item Axios interceptor catches 401 responses and handles session cleanup
\end{itemize}

\subsection{Password Strength Indicator}

During registration, a 4-bar visual indicator shows password strength:
\begin{itemize}[nosep]
    \item Bar 1: Minimum 6 characters
    \item Bar 2: Minimum 10 characters
    \item Bar 3: Contains uppercase letter
    \item Bar 4: Contains special character
\end{itemize}

\subsection{Responsive Design}

\begin{itemize}[nosep]
    \item Mobile: Single-column layout with adapted controls
    \item Desktop: Three-column Kanban grid
    \item NavBar buttons adapt (icon-only on mobile, text + icon on desktop)
\end{itemize}

\subsection{Confirm Modal for Destructive Actions}

Deleting a task requires confirmation through a modal with a warning icon, a descriptive message, and Cancel/Delete buttons.

\subsection{Dynamic Tags System}

Tasks support tags that are entered dynamically (press Enter or comma to add). Tags are displayed as blue pills on task cards and stored as comma-separated values in the database.

\section{Authentication Flow}

\begin{enumerate}[nosep]
    \item \textbf{Registration}\\
    Client sends \texttt{POST /api/auth/register} with \texttt{\{username, email, password\}}.\\
    Server hashes the password with bcrypt, inserts into the database, and returns \texttt{201 Created}.

    \item \textbf{Login}\\
    Client sends \texttt{POST /api/auth/login} with \texttt{\{email, password\}}.\\
    Server verifies credentials (bcrypt compare), generates a JWT token \texttt{\{userId, email\}} with 24h expiry, and returns \texttt{\{token, userId, username\}}.\\
    Client stores the token in localStorage and redirects to \texttt{/dashboard}.

    \item \textbf{Authenticated Requests}\\
    The Axios request interceptor attaches the \texttt{Authorization: Bearer <token>} header.\\
    The server middleware verifies the JWT, attaches \texttt{req.user}, and allows or denies access.

    \item \textbf{Session Expiry}\\
    Client parses the JWT payload to extract the expiry timestamp.\\
    A warning toast is scheduled 5 minutes before expiry.\\
    Auto-logout triggers when the token expires, clearing storage and redirecting to \texttt{/login}.

    \item \textbf{Logout}\\
    Client clears localStorage and sessionStorage, then redirects to \texttt{/login}.
\end{enumerate}

\vspace{0.5em}
\textbf{Security measures:}
\begin{itemize}[nosep]
    \item Passwords hashed with bcrypt (never stored in plain text)
    \item JWT tokens with 24-hour expiration
    \item All SQL queries use parameterized placeholders (SQL injection prevention)
    \item Task ownership enforcement (users can only access their own tasks)
    \item Client-side 401 interception and automatic session cleanup
\end{itemize}

\section{Conclusion}

Milestone~2 is fully completed. The front-end has been developed using \textbf{Vue.js~3} with the \textbf{Composition API}, styled with \textbf{Tailwind CSS}, and fully integrated with the Express.js back-end API.
